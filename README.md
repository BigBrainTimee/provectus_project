# Claude Code Usage Analytics Platform

## Project Overview

This project implements an end-to-end analytics platform that processes telemetry data from Claude Code sessions and transforms it into actionable insights about developer behavior and usage patterns.

The system ingests raw telemetry event streams, processes and validates the data, stores it in a structured database, and provides analytics through an interactive dashboard.

The goal of this project is to demonstrate capabilities in:

* Data processing pipelines
* Database schema design
* Analytics and insight generation
* Data visualization
* AI-assisted software development

The platform follows an **AI-first development approach**, leveraging LLM tools such as Cursor and ChatGPT to accelerate development and improve implementation efficiency.

---

# System Architecture

The system is organized into several layers:

## 1. Data Ingestion

Raw telemetry data is loaded from JSON and CSV datasets. These files simulate telemetry streams generated by developer sessions in Claude Code.

## 2. Data Validation

Incoming telemetry events are validated to ensure:

* Correct data structure
* Required fields are present
* Timestamps and metadata are valid
* Incorrect or malformed records are safely handled

## 3. Data Transformation

Raw events are normalized and structured into a format suitable for analytics and database storage.

## 4. Data Storage

Processed data is stored in a **SQLite database** using a normalized schema with optimized indexes for efficient analytical queries.

Main database tables include:

* **employees** – user metadata such as role and project type
* **sessions** – aggregated information about development sessions
* **events** – individual telemetry events
* **token_usage** – token consumption data used for analytics

## 5. Analytics Layer

An `AnalyticsService` module performs analytical queries on the stored data to generate insights such as:

* Token consumption trends
* Developer activity patterns
* Session statistics
* Tool usage behavior

## 6. Visualization Layer

An interactive **Streamlit dashboard** presents analytics results through charts, metrics, and tables.

The dashboard allows users to explore telemetry data and understand usage patterns across different roles, sessions, and tools.

# Installation

Clone the repository:

```bash
git clone <repository-url>
cd <repository-folder>
```

Install required dependencies:

```bash
pip install -r requirements.txt
```

---

# Running the Data Pipeline

Before launching the dashboard, the telemetry dataset must be processed and inserted into the database.

Run the data processing pipeline:

```bash
python process_data.py
```

This step performs the following tasks:

1. Loads the raw telemetry dataset
2. Validates the structure of telemetry events
3. Cleans and transforms the data
4. Inserts processed data into the SQLite database

---

# Running the Dashboard

Start the analytics dashboard using Streamlit:

```bash
streamlit run dashboard.py
```

After launching, the dashboard will open in your browser and display interactive analytics visualizations.

# Dashboard Overview

The dashboard includes multiple sections:

### Overview

High-level metrics including total tokens, sessions, events, and users.

### Token Usage Analytics

Analysis of token consumption by role and project type.

### Usage Patterns

Insights into peak activity hours, event distribution, and common developer actions.

### Session Insights

Statistics on session duration, events per session, and activity levels.

### User Activity

Visualization of the most active users and session distribution.
